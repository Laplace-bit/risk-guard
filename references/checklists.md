# Phase 1: Checklists (检查清单)

Checklists are the most evidence-based method for reducing unexpected failures.
Surgical mortality dropped 47% with a simple checklist (Gawande). Pilots use them
before every landing. The power is in forced completeness — no known pitfall
should slip through.

## How to Use

For each item: check ✅ (confirmed safe), ⚠️ (risk exists, noted), or 🔴 (blocks execution).
Any 🔴 item MUST be resolved before proceeding.

---

## Universal Checklist (适用于所有计划)

### 基本确认

- [ ] **目标明确吗？** — 能用一句话说清成功标准吗？
- [ ] **回退方案存在吗？** — 如果出问题，能回到之前的状态吗？
- [ ] **谁在执行？** — 每一步有明确的负责人吗？
- [ ] **时间估计靠谱吗？** — 是否用了参考类预测而非内部预估？
- [ ] **依赖确认了吗？** — 所有人/服务/资源都确认可用了吗？
- [ ] **沟通到位了吗？** — 受影响的人知道这件事要发生吗？

### 常见认知陷阱

- [ ] **规划谬误检查** — 实际时间是否通常是估计的2-3倍？
- [ ] **锚定检查** — 时间/成本估计是否被第一个数字锚定了？
- [ ] **可用性启发检查** — 是否只因为某个风险最近发生过就高估了它？
- [ ] **确认偏误检查** — 是否只搜索了支持计划的证据？找3个反对理由
- [ ] **沉没成本检查** — 是否因为已经投入了就继续？
- [ ] **乐观偏误检查** — "这次会不一样"有证据支持吗？

### 通信与协调

- [ ] **通知了所有受影响方吗？** — 下游依赖方、相关人、管理层
- [ ] **有应急联系方式吗？** — 如果出问题，第一个找谁？
- [ ] **有状态同步机制吗？** — 执行过程中怎么让相关人知道进展？

## Safety Scenario Checklists (安全场景清单)

> These are the PRIMARY checklists for this skill. Use the one that matches
> the user's scenario. If multiple apply, combine them.

### ✈️ 出行与交通 (Travel & Mobility)

- [ ] **天气和路况查了吗？** — 不要问用户，先自己搜
- [ ] **回程方案有吗？** — 第一计划失败后的备选交通
- [ ] **证件/票务确认了吗？** — 身份证、护照、机票、车票
- [ ] **有人在目的地等你并知道你的时间表吗？**
- [ ] **手机电量充足吗？** — 充电宝、车充
- [ ] **贵重物品是否最少化？** — 不必要的不带
- [ ] **如果是别人开车：司机是否休息好、清醒、熟悉路线？**
- [ ] **车辆状况如何？** — 轮胎、刹车、近期保养
- [ ] **如果是自己开车：是否有同行者可以替换？**

### 👤 陌生人见面 (Stranger Meeting)

- [ ] **对方身份是否经过独立验证？** — 不只是聊天软件上的自述
- [ ] **第一次见面是否在公共场所？** — 咖啡厅、商场等，而非对方住所或偏僻地点
- [ ] **是否有自己的来回交通？** — 不依赖对方接送
- [ ] **是否有人知道你在哪、和谁见面、何时应该报平安？**
- [ ] **是否有压力喝酒、换地方、或临时改变计划？**
- [ ] **手机是否满电？** — 并提前告知朋友见面细节

### 💰 交易与支付 (Transaction & Payment)

- [ ] **支付方式是否可逆？** — 信用卡可退款，现金/转账/加密货币难追回
- [ ] **对方身份是否通过独立渠道验证？** — 不只是对方提供的链接或信息
- [ ] **是否有时间压力或要求保密？** — "必须现在付"、"别告诉别人"是红旗
- [ ] **金额是否与市场价相符？** — 过低或过高都要警惕
- [ ] **合同/收据/凭证是否齐全？**
- [ ] **是否有冷静期可以撤回？**

### 🏕️ 户外与环境暴露 (Outdoor & Environment)

- [ ] **天气预报查了吗？** — 包括返回时的天气，不要只看出发时
- [ ] **路线难度和返回时间确认了吗？** — 不要只考虑去程
- [ ] **水源、遮阳、信号覆盖、救援路线是否确认？**
- [ ] **是否有同伴？** — 同伴是否有能力帮助你？
- [ ] **是否告知了他人你的路线和预计返回时间？**
- [ ] **应急装备是否齐全？** — 急救包、地图/GPS、保暖衣物
- [ ] **是否有天气或地形恶化的备选方案？**

### 🌙 夜间出行与独处 (Nightlife & Isolated Time)

- [ ] **是否有人知道你的行踪和预计回家时间？**
- [ ] **是否在光线充足、有人流的区域？**
- [ ] **是否有可靠的回家交通？** — 不依赖临时搭车
- [ ] **手机是否满电？** — 是否设置了紧急联系人和位置共享
- [ ] **是否与同伴一起？** — 如果分开，是否有集合点和时间
- [ ] **饮酒是否可控？** — 是否始终能看到自己的杯子

### 🏥 健康敏感活动 (Health-Sensitive Activity)

- [ ] **是否有怀孕可能性、术后恢复、慢性病、或用药因素？**
- [ ] **是否有特殊体质？** — G6PD缺乏、甲状腺疾病、免疫抑制、金属过敏、严重过敏、光敏感、凝血障碍、癫痫、电磁敏感/植入设备
- [ ] **活动是否涉及化学品暴露、感染风险、辐射暴露、长时间站立/行走、极端温度？**
- [ ] **现场是否有急救设施和医疗支持？** — 最近医院/诊所多远？
- [ ] **是否可以随时中断离开？** — 不受流程或社会压力阻碍
- [ ] **是否有医生的限制或建议需要遵守？**
- [ ] **是否携带了必要的药物和医疗信息？**
- [ ] **特殊体质 + 环境暴露是否已交叉检查？** — 见下方体质-暴露交互表
- [ ] **如果是育龄女性：是否确认未怀孕？** — 涉及辐射/化学致畸物/活疫苗时必须确认
- [ ] **如果是备孕期：男女双方的暴露是否已评估？** — 生殖毒性不只影响女性
- [ ] **如果是哺乳期：暴露是否可通过乳汁传递？** — 有机溶剂、重金属、某些药物
- [ ] **如果是孕早期：是否在器官发生期（0-12周）？** — 致畸风险最高阶段

**体质-暴露交互检查：**

| 体质 | 必须排查的暴露 | 典型场景 |
|------|----------------|----------|
| G6PD缺乏 | 樟脑/萘、磺胺类、蚕豆、伯氨喹、亚甲蓝 | 家政樟脑丸、疟区旅行用药、某些食品 |
| 甲状腺疾病 | 电离辐射、含碘造影剂 | 医学影像、核设施、工业辐射 |
| 哮喘/COPD | 粉尘、VOC、化学烟气、冷空气 | 工厂、装修、实验室、冬季户外 |
| 免疫抑制 | 生物暴露、活疫苗、霉菌 | 医院、工地、人群聚集、旅行 |
| 金属过敏 | 工业化学品、金属粉尘、合金 | 工厂、金属加工、首饰制作 |
| 孕期/可能怀孕 | 辐射、有机溶剂、重金属、内分泌干扰物、活疫苗 | 化学实验室、工厂、医疗影像、新装修 |
| 光敏感(SLE等) | 紫外线、光敏药物 | 户外、海滩、高海拔 |
| 凝血障碍 | 创伤环境、抗凝药物 | 极限运动、手术、牙科 |
| 癫痫 | 开放水域、高空、闪烁光源 | 游泳、蹦极、VR体验 |
| 电磁敏感/植入设备 | 强电磁场、MRI | 核磁共振、变电站、工业感应加热 |
| 严重过敏 | 食物、昆虫、乳胶、特定化学 | 户外、新环境餐饮、医疗场景 |

### 👶 看护与依赖保护 (Caregiving & Dependent)

- [ ] **被看护人的日常需求在坏情况下也能满足吗？** — 不只考虑好天气
- [ ] **看护者中断时有备用人选吗？**
- [ ] **紧急联系路径是否确认？** — 不只是一条
- [ ] **药物时间表是否可靠？** — 延误的后果是什么
- [ ] **交通是否随时可用？** — 不依赖单一方式

### 🏠 房屋与物业查看 (Housing & Property)

- [ ] **房源和房东身份是否独立验证？** — 不只是平台上的描述
- [ ] **是否在要求押金之前看了房？** — 先付后看是典型骗局
- [ ] **看房是否在白天、有人流的地方？**
- [ ] **是否有人陪同？**
- [ ] **是否被要求提供敏感证件或大量个人信息？**

### 💻 数字诈骗与钓鱼 (Digital Fraud & Phishing)

- [ ] **是你主动联系对方，还是对方主动联系你？** — 主动联系你是红旗
- [ ] **是否要求提供密码、验证码、身份证件或紧急转账？**
- [ ] **是否有时间压力或威胁？** — "你的账户将被冻结"、"限时优惠"
- [ ] **是否可以通过独立渠道验证对方身份？** — 打官方电话，不点链接
- [ ] **链接或域名是否与官方一致？** — 注意细微拼写差异
- [ ] **支付方式是否可逆？** — 要求加密货币、礼品卡、现金汇款是红旗

### 🏢 商务出差 (Business Trip)

- [ ] **目的地的安全状况是否确认？** — 旅行警告、近期事件
- [ ] **住宿安全性如何？** — 酒店评分、区域安全、24小时前台
- [ ] **是否有本地紧急联系方式和最近医院位置？**
- [ ] **护照/签证/保险是否齐全？**
- [ ] **是否携带贵重设备或大量现金？** — 是否有安全存放处
- [ ] **是否有语言障碍？** — 是否有翻译或本地联系人
- [ ] **是否有人知道完整行程和每日报平安时间？**

### 🌐 线上转线下 (Online-to-Offline)

- [ ] **线上身份是否经过视频/语音验证？** — 不只是文字聊天
- [ ] **线下见面是否在公共场所？**
- [ ] **是否有人知道你的行踪和见面细节？**
- [ ] **对方是否有紧迫感、要求金钱或文件？**
- [ ] **是否可以随时中断不依赖对方？**

### 🏭 工地/工厂/实验室参观 (Workplace & Site Visit)

- [ ] **现场有哪些危害？** — 化学品、粉尘、噪音、高温、辐射、生物危害、高空作业
- [ ] **你是否有特殊体质与上述危害交互？** — 哮喘+粉尘、甲状腺+辐射、免疫抑制+生物危害、金属过敏+化学品、孕期+有机溶剂
- [ ] **PPE 是否确认到位，不只是假设？** — 安全帽、防护服、护目镜、呼吸器
- [ ] **是否有入场安全培训/指引？** — 不是自己随便走
- [ ] **紧急出口和集合点在哪里？** — 是否有人告知
- [ ] **是否可以随时中断离开？** — 不受流程或面子压力阻碍
- [ ] **现场医疗资源在哪？** — 急救箱、急救员、最近医院距离
- [ ] **预计多少步行/站立/搬运/攀爬？**
- [ ] **是否有监护人/陪同人员？** — 不应独自在陌生危险区域活动

### 💼 面试与入职 (Job Interview & Onboarding)

- [ ] **面试地点是否在正规办公场所？** — 不是私人住宅、酒店房间、偏僻地点
- [ ] **公司是否可独立验证？** — 工商注册、官网、口碑评价
- [ ] **是否有人知道你去哪面试、何时应报平安？**
- [ ] **是否要求交押金、培训费、证件原件？** — 正规公司不会
- [ ] **是否有自己的交通往返？** — 不依赖对方接送
- [ ] **合同条款是否仔细阅读？** — 特别注意竞业限制、薪资结构、试用期条款
- [ ] **是否被要求提供银行密码、手机、社交账号？** — 绝对红旗

### 🏥 就医与医疗决策 (Medical Visit & Decision)

- [ ] **医院/诊所是否有正规资质？** — 不是"朋友介绍的私人诊所"
- [ ] **是否已获取第二诊疗意见？** — 特别是对手术、重大治疗
- [ ] **治疗方案的风险和替代方案是否解释清楚？**
- [ ] **手术/麻醉是否有知情同意书？**
- [ ] **是否有术后照护和应急方案？**
- [ ] **费用是否透明？** — 是否有隐藏费用、过度检查
- [ ] **是否可以随时转院或换医生？** — 不被锁定

### 🌊 自然灾害应对 (Natural Disaster Response)

- [ ] **当前预警级别和预计影响是什么？** — 不要问用户，先自己搜
- [ ] **是否需要撤离？** — 撤离路线和避难点是否确认
- [ ] **应急物资是否齐全？** — 水、食物、药品、手电、充电器、现金
- [ ] **家人/同伴的联络和集合方案是否确认？**
- [ ] **是否有人在灾害中更脆弱？** — 老人、儿童、孕妇、残障、病人
- [ ] **通信中断时的备用联络方式？** — 不只依赖手机
- [ ] **是否可以推迟不必要的出行？** — 大多数灾害风险来自"非要出门"

### 🏠 家庭与居家安全 (Home & Family Safety)

- [ ] **独居老人/儿童是否有人定期查看？** — 每日至少一次联络
- [ ] **燃气/电器/取暖设备是否安全？** — 是否有漏气保护、过载保护
- [ ] **烟雾报警器和灭火器是否正常？**
- [ ] **是否有陌生人可以进入家中？** — 家政、维修、快递
- [ ] **紧急联系人是否老人/儿童也能操作？** — 不仅是拨电话
- [ ] **是否有备用钥匙/锁门方案？** — 不被锁在外面或里面
- [ ] **药品/化学品/尖锐物品是否远离儿童？**

### 🤝 求职与兼职陷阱 (Job Scam & Exploitation)

- [ ] **招聘信息是否通过正规平台发布？** — 不是微信群、短信、路边小广告
- [ ] **是否要求先交费（押金/培训/服装/体检）？** — 正规工作不收费
- [ ] **工作地点和内容是否清晰？** — 模糊描述是传销/诈骗信号
- [ ] **是否要求扣押身份证、银行卡、手机？** — 绝对红旗
- [ ] **薪酬是否远高于市场水平？** — 过高必有陷阱
- [ ] **是否限制人身自由或通讯？** — 不允许离开、不允许联系家人
- [ ] **是否有书面合同？** — 口头承诺无效

### 🏋️ 运动与健身 (Sports & Fitness)

- [ ] **运动强度是否匹配当前身体状况？** — 不要从零直接上高强度
- [ ] **是否有特殊体质需注意？** — 哮喘（冷空气/氯气）、心脏病（高强度）、凝血障碍（创伤性运动）、癫痫（开放水域/高空）、严重过敏（昆虫/远离急诊）
- [ ] **是否有热身和逐步加量？**
- [ ] **是否有教练/有经验者陪同？** — 特别是非熟悉运动
- [ ] **环境条件是否安全？** — 高温、高湿、高海拔、深水、极端寒冷
- [ ] **是否有急救准备？** — AED、急救人员、最近医院
- [ ] **极限运动是否有专业指导和保险？**
- [ ] **是否有基础疾病需要限制运动？** — 心脏病、哮喘、怀孕等

### 🔧 家政与装修 (Home Service & Renovation)

- [ ] **服务人员身份是否验证？** — 平台认证、身份登记、照片确认
- [ ] **是否有人在家陪同？** — 特别是独居时
- [ ] **贵重物品是否收好？** — 不要让陌生人接触现金、证件、首饰
- [ ] **合同和报价是否书面确认？** — 口头报价无效
- [ ] **工期和付款方式是否明确？** — 不预付全款
- [ ] **材料是否环保安全？** — 甲醛、涂料、粉尘对儿童/孕妇的影响
- [ ] **特殊体质是否与装修材料交互？** — G6PD缺乏+樟脑丸、哮喘+粉尘/VOC、孕期/儿童+甲醛、免疫抑制+霉菌、化学敏感+清洁剂/粘合剂
- [ ] **是否有售后/质保条款？** — 出问题找谁

## Technical Scenario Checklists (技术场景清单)

> Use these when the Full Review is triggered for a technical operation
> (deployment, migration, etc.). These are SECONDARY to the safety checklists above.

### 🚀 部署 (Deploy)

- [ ] **回滚方案测试过吗？** — 不仅写了，还在类似环境跑过
- [ ] **数据库迁移可逆吗？** — 特别是 schema 变更
- [ ] **功能开关就位了吗？** — 能否一键关掉新功能？
- [ ] **监控和告警设好了吗？** — 关键指标的基线是什么？
- [ ] **灰度/金丝雀发布计划有吗？** — 不是全量上线
- [ ] **依赖服务版本兼容吗？** — API 变更的消费者都更新了吗？
- [ ] **缓存策略确认了吗？** — 旧缓存会导致数据不一致吗？
- [ ] **健康检查端点更新了吗？** — 负载均衡器能正确判断新版本吗？

### 🔄 迁移 (Migration)

- [ ] **源数据完整性验证了吗？** — 迁移前做数据校验
- [ ] **双写期间数据一致性怎么保证？** — 新旧系统同时写入的策略
- [ ] **迁移脚本在大数据量下测过吗？** — 1K行能跑不代表200M行能跑
- [ ] **外键和约束都记录了吗？** — 未文档化的 FK 是定时炸弹
- [ ] **迁移失败时的回滚点在哪？** — 每一步都有回滚点，不只是整体回滚
- [ ] **停机窗口确认了吗？** — 谁批准的？多长？超时怎么办？
- [ ] **数据对比验证方案有吗？** — 迁移后怎么确认数据一致？

### 🗄️ 数据操作 (Data Operations)

- [ ] **备份存在且已验证可恢复吗？** — 不只是备份了，是恢复过
- [ ] **操作在事务中吗？** — 失败能自动回滚
- [ ] **批量操作有限流吗？** — 大批量INSERT/UPDATE是否分批+间隔
- [ ] **锁影响评估了吗？** — 是否会阻塞其他关键操作
- [ ] **数据隐私合规了吗？** — 删除/导出/脱敏是否符合法规要求
- [ ] **幂等性确认了吗？** — 重复执行不会造成数据错误
- [ ] **测试数据和生产数据隔离了吗？** — 不应该用生产数据测试

### 🔗 集成 (Integration)

- [ ] **API 契约双方确认了吗？** — 不是"文档说能行"，是实测过
- [ ] **超时和重试策略对齐了吗？** — 下游超时必须长于上游
- [ ] **限流和配额确认了吗？** — 对方API有调用频率限制吗？
- [ ] **错误处理和降级策略有吗？** — 对方挂了怎么办？
- [ ] **认证和授权配置了吗？** — Token、密钥、权限都到位了
- [ ] **网络可达性确认了吗？** — 防火墙、代理、VPN、跨环境
- [ ] **数据格式兼容吗？** — 枚举值、空值处理、时区、编码

### 📦 发布 (Release)

- [ ] **版本号和变更日志更新了吗？** — 用户能看到改了什么
- [ ] **breaking changes 通知了吗？** — API变更、配置变更、行为变更
- [ ] **文档和代码同步了吗？** — API文档、README、示例代码
- [ ] **已知问题记录了吗？** — 不影响发布但用户应该知道的限制
- [ ] **发布顺序对吗？** — 先API后消费者？先后端后前端？
- [ ] **回滚发布流程确认了吗？** — 不是回滚代码，是回滚已发布的版本
- [ ] **安全扫描通过了吗？** — 依赖漏洞、密钥泄露

### ⚙️ 配置变更 (Config Change)

- [ ] **配置变更的爆炸半径评估了吗？** — 影响范围有多大？
- [ ] **能热更新还是需要重启？** — 重启会中断什么？
- [ ] **旧值备份了吗？** — 能快速恢复吗？
- [ ] **环境一致性确认了吗？** — 所有环境（dev/staging/prod）同步确认
- [ ] **配置验证机制有吗？** — 格式错误能被拒绝而不是静默生效
- [ ] **变更审计日志有吗？** — 谁在什么时候改了什么

## Anti-Checklist (这些不是检查清单)

以下情况说明你可能在错误地使用清单：

- ❌ 用清单代替思考 — 清单确保不遗漏已知问题，不能发现未知问题
- ❌ 清单太长 — 超过30项的清单会让人跳着看，反而漏掉重点
- ❌ 只勾不查 — 每个✅都应该有证据支撑，不是"应该没问题"

**清单之后的Phase 2-6正是为了弥补清单的盲区。**
